## 传统爬虫的脆弱性

CSS 选择器和 XPath 的问题是：**页面结构一变就崩**。一个 class 重命名、DOM 层级调整，精心写的选择器就废了。

```python
# 传统方式：选择器一改就崩
price = soup.select_one("span.product-price--current").text
```

## LLM 提取：把问题从"选哪里"变成"要什么"

AI 爬虫的思路是：**LLM 理解页面内容，直接提取你要的信息**。不管 DOM 结构怎么变，语义不变就能提。

```python
from crawl4ai import AsyncWebCrawler

schema = {
    "name": "Products",
    "baseSelector": "div.product-card",
    "fields": [
        {"name": "title", "selector": "h2", "type": "text"},
        {"name": "price", "selector": ".price", "type": "text"},
    ]
}

async with AsyncWebCrawler() as crawler:
    result = await crawler.crawl(url, extraction_strategy=schema)
```

## Crawl4AI：专为 AI 设计的爬虫框架

Crawl4AI 是 2026 年最受关注的爬虫工具之一，在 GitHub 上已有超过 60K star。它的核心设计目标就是"为 LLM 准备数据"：

**一键转 Markdown**——爬下来的页面自动转成干净的 Markdown，LLM 最喜欢的输入格式：

```python
result = await crawler.crawl(url)
print(result.markdown)  # 干净的 Markdown 输出
```

**LLM 提取策略**——直接给 Pydantic 模型，让 LLM 填充：

```python
from pydantic import BaseModel
from crawl4ai.extraction_strategy import LLMExtractionStrategy

class Product(BaseModel):
    title: str
    price: float
    description: str
    rating: float | None = None

strategy = LLMExtractionStrategy(
    llm_config={"provider": "openai/gpt-4"},
    schema=Product.model_json_schema(),
    instruction="提取页面中的商品信息",
)
```

**智能爬取策略**——支持 BFS、DFS 和 BestFirst 三种策略：

```python
config = CrawlerRunConfig(
    extraction_strategy=strategy,
    deep_crawl_strategy="best_first",
    max_pages=50,
)
```

## AI 爬虫的最佳实践

**更好的工作流**是把爬取和提取分开：

```
抓取器（Crawlbase / Scrapling）→ 转 Markdown → LLM 提取结构化数据
```

这样抓取层关注反爬和渲染，提取层关注语义理解，各司其职。

```python
# 1. 抓取层：获取干净页面内容
page_text = fetch_with_anti_bot(url)

# 2. 提取层：LLM 理解内容
prompt = f"""
从以下文章中提取：标题、发布日期、作者、正文摘要。
返回 JSON。

{page_text[:8000]}
"""

result = llm.complete(prompt)
```

## Scrapling：自适应选择器

Scrapling 是另一个实用工具，它会在首次运行时保存元素的"结构指纹"，页面改版后能根据相似度重新定位元素：

```python
from scrapling.fetchers import Fetcher

page = Fetcher.get(url, selector_config={"adaptive": True})
# 首次运行：保存指纹
title = page.css("h1.product-title", auto_save=True)
# 页面改版后：自适应匹配
title = page.css("h1.product-title", adaptive=True)
```

## 总结

AI + 爬虫不是取代传统爬虫，而是增加了一个高维度的工具。**CSS 选择器适合结构稳定的页面，LLM 提取适合页面频繁变动或结构复杂的场景。** 两者结合，才能构建健壮的生产级爬虫。
