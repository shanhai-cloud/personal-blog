## 架构总览

```
客户端 → Spring Cloud Gateway（集成 Sentinel）
                    ↓
      Nacos 注册中心 + 配置中心
                    ↓
      ┌──────┬──────┬──────┐
      │ 用户  │ 订单  │ 商品  │  ← 微服务实例
      └──────┴──────┴──────┘
                    ↓
      Redis + MySQL + RocketMQ
```

## Nacos：注册中心与配置中心

### 安装 Nacos

```bash
docker run -d --name nacos -p 8848:8848 nacos/nacos-server
```

### 服务注册

```yml
# application.yml
spring:
  cloud:
    nacos:
      discovery:
        server-addr: localhost:8848
```

### 配置中心

把配置文件从项目中移到 Nacos，支持动态刷新：

```yml
# bootstrap.yml
spring:
  cloud:
    nacos:
      config:
        server-addr: localhost:8848
        file-extension: yaml
```

```java
@RefreshScope
@RestController
public class ConfigController {
    @Value("${app.config}")
    private String config;  // 修改 Nacos 中的配置后自动刷新
}
```

## Spring Cloud Gateway：API 网关

```yml
spring:
  cloud:
    gateway:
      routes:
        - id: user-service
          uri: lb://user-service
          predicates:
            - Path=/api/users/**
          filters:
            - StripPrefix=1
```

从 Spring 6 开始，服务间调用推荐用原生的 `HttpExchange` 替代 OpenFeign：

```java
@HttpExchange("/users")
public interface UserClient {
    @GetExchange("/{id}")
    User getUser(@PathVariable Long id);
}
```

配合负载均衡：

```java
@Bean
@LoadBalanced
public WebClient.Builder webClientBuilder() {
    return WebClient.builder();
}
```

## Sentinel：流量控制与熔断降级

### 网关层限流

```yml
spring:
  cloud:
    sentinel:
      filter:
        enabled: true
      datasource:
        ds:
          nacos:
            server-addr: localhost:8848
            dataId: sentinel-gateway-rules
```

### 服务间熔断

```java
@SentinelResource(value = "getUser", fallback = "getUserFallback")
public User getUser(Long id) {
    return userClient.getUser(id);
}

public User getUserFallback(Long id, Throwable t) {
    return new User(id, "默认用户");  // 降级兜底
}
```

### 规则持久化

Sentinel 的限流规则默认存在内存里，重启就丢了。务必用 Nacos 持久化：

```java
@PostConstruct
public void initRules() {
    var rules = new ArrayList<FlowRule>();
    var rule = new FlowRule();
    rule.setResource("getUser");
    rule.setGrade(RuleConstant.FLOW_GRADE_QPS);
    rule.setCount(100);  // 每秒最多 100 个请求
    rules.add(rule);
    FlowRuleManager.loadRules(rules);
}
```

## 总结

Spring Cloud Alibaba 三件套（Nacos + Gateway + Sentinel）是目前国内微服务的主流选型。核心关注点就三个：

1. **注册中心管服务发现**——Nacos 还兼任配置中心，一笔投入双重收益
2. **网关管流量入口**——路由转发 + 统一鉴权 + 限流
3. **Sentinel 管服务质量**——限流 + 熔断 + 降级，保证不雪崩
