## 为什么用 Java 做爬虫

Python 是爬虫领域的常客，但 Java 在大型爬虫项目中有不可替代的优势：强类型系统让数据结构更可控，JVM 的成熟线程模型让并发抓取更稳定，加上 Spring 等生态的加持，适合构建生产级的抓取系统。

## 第一步：Java HttpClient 发送请求

从 Java 11 开始，JDK 内置了 `java.net.http.HttpClient`，无需任何第三方依赖即可发送 HTTP 请求：

```java
var client = HttpClient.newHttpClient();
var request = HttpRequest.newBuilder()
    .uri(URI.create("https://example.com"))
    .header("User-Agent", "Mozilla/5.0")
    .GET()
    .build();

var response = client.send(request, HttpResponse.BodyHandlers.ofString());
System.out.println(response.body());
```

### 封装重试和超时

生产环境的请求必须有超时和重试：

```java
var client = HttpClient.newBuilder()
    .connectTimeout(Duration.ofSeconds(10))
    .build();

// 简单的重试封装
int retries = 3;
while (retries-- > 0) {
    try {
        var resp = client.send(request, BodyHandlers.ofString());
        if (resp.statusCode() == 200) return resp.body();
    } catch (IOException e) {
        if (retries == 0) throw e;
        Thread.sleep(1000);
    }
}
```

## 第二步：Jsoup 解析 HTML

请求拿到 HTML 后，Jsoup 是最主流的解析库。它支持 CSS Selector 和 DOM 遍历，API 设计得相当直观：

```xml
<dependency>
    <groupId>org.jsoup</groupId>
    <artifactId>jsoup</artifactId>
    <version>1.18.1</version>
</dependency>
```

### 解析文章列表

```java
var doc = Jsoup.parse(html);
var title = doc.select("h1.article-title").text();
var content = doc.select("div.article-content").html();
var links = doc.select("a[href]").stream()
    .map(a -> a.attr("abs:href"))
    .toList();
```

### 处理相对路径

Jsoup 提供了 `abs:` 前缀属性提取，自动解析相对路径为绝对 URL，这个细节很实用：

```java
var absUrl = link.attr("abs:href");  // 自动拼接 base URI
```

## 第三步：Selenium 处理动态页面

当目标页面依赖 JavaScript 渲染时，仅靠 HTTP 请求拿不到真实内容。这时需要 Selenium 控制真实浏览器：

```xml
<dependency>
    <groupId>org.seleniumhq.selenium</groupId>
    <artifactId>selenium-java</artifactId>
    <version>4.27.0</version>
</dependency>
```

### 无头浏览器配置

```java
var options = new ChromeOptions();
options.addArguments("--headless", "--disable-gpu", "--no-sandbox");
var driver = new ChromeDriver(options);

driver.get("https://example.com");
// 等待 JS 渲染完成
driver.manage().timeouts().implicitlyWait(Duration.ofSeconds(10));

var pageSource = driver.getPageSource();
driver.quit();
```

用工厂模式管理 WebDriver 实例，避免频繁启停：

```java
public class WebDriverFactory {
    private static final ThreadLocal<WebDriver> drivers = new ThreadLocal<>();

    public static WebDriver getDriver() {
        if (drivers.get() == null) {
            var options = new ChromeOptions();
            options.addArguments("--headless");
            drivers.set(new ChromeDriver(options));
        }
        return drivers.get();
    }

    public static void close() {
        if (drivers.get() != null) {
            drivers.get().quit();
            drivers.remove();
        }
    }
}
```

## 第四步：多线程并发抓取

单线程抓取效率太低，用线程池来加速：

```java
var executor = Executors.newFixedThreadPool(8);
var tasks = urls.stream()
    .map(url -> (Callable<Result>) () -> fetchAndParse(url))
    .toList();

var results = executor.invokeAll(tasks);
executor.shutdown();
```

### Java 21 虚拟线程方案

如果使用 Java 21+，虚拟线程能让并发模型更简洁：

```java
try (var executor = Executors.newVirtualThreadPerTaskExecutor()) {
    var futures = urls.stream()
        .map(url -> executor.submit(() -> fetchAndParse(url)))
        .toList();
    for (var future : futures) {
        results.add(future.get());
    }
}
```

## 第五步：反爬应对策略

实际抓取中一定会遇到反爬措施，常见的应对手段：

| 反爬手段 | 应对方案 |
|----------|----------|
| IP 封禁 | 代理池（快代理、亿牛云），轮换出口 IP |
| User-Agent 检测 | 维护 UA 池，随机轮换 |
| 请求频率限制 | RateLimiter 控制 QPS，随机休眠 |
| 动态渲染 | Selenium / Playwright 无头浏览器 |
| 验证码 | 接入打码平台，或优化策略减少触发 |

### 请求限速

```java
// Guava RateLimiter，每秒最多 5 个请求
var limiter = RateLimiter.create(5.0);
limiter.acquire();  // 阻塞直到获取许可
client.send(request, BodyHandlers.ofString());
```

## 第六步：框架选择

如果不想从零搭建，可以直接使用成熟的爬虫框架：

**WebMagic**：国产开源框架，四大核心组件（Downloader、Processor、Pipeline、Scheduler），支持注解式声明和分布式扩展，适合 Java 开发者快速上手。

**Crawl4j**：专注高效抓取，内置布隆过滤器做 URL 去重，支持分布式部署，性能表现突出。

```java
// WebMagic 示例
public class BlogProcessor implements PageProcessor {
    private Site site = Site.me().setRetryTimes(3).setSleepTime(1000);

    @Override
    public void process(Page page) {
        page.putField("title", page.getHtml().xpath("//h1/text()"));
        page.putField("content", page.getHtml().css("div.content").get());
        page.addTargetRequests(page.getHtml().links().regex("https://blog/.*").all());
    }

    @Override
    public Site getSite() { return site; }

    public static void main(String[] args) {
        Spider.create(new BlogProcessor())
            .addUrl("https://example.com")
            .thread(8)
            .run();
    }
}
```

## 合规提醒

爬虫不是法外之地，几点原则要遵守：

1. **遵守 robots.txt** — 尊重目标网站的抓取规则
2. **控制频率** — 不要对目标服务器造成压力
3. **数据使用** — 抓取的数据要注意版权和隐私合规
4. **身份标识** — 设置有意义的 User-Agent，标明爬虫身份

## 总结

Java 爬虫的技术栈可以概括为：

**静态页面**：HttpClient + Jsoup → 最轻量，适合大部分场景
**动态页面**：Selenium / Playwright → 处理 JS 渲染
**大规模**：WebMagic / Crawl4j + 虚拟线程 → 分布式爬取

选择什么工具取决于你的目标网站复杂度和数据规模。从 HttpClient 开始，按需升级，不要一开始就上重型框架。
